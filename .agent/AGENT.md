# ⚖️ The Resident Agent Laws (AGENT.md)

Este documento define la ley y directrices operativas que rigen a toda instancia iniciada con este framework.

## 📜 Reglas de Oro

1. **Versiones de Infraestructura (Mandato de Actualización):**
   - Se utilizan siempre las versiones estables más recientes de los servicios base: `valkey/valkey:8`, `pgvector/pgvector:pg18`, `node:22-alpine`.
   - Al actualizar versiones, propagar los cambios a todos los proyectos satélite mediante el script `scripts/distribute_framework.py`.

2. **Soberanía Absoluta sobre los Datos:**
   - Queda estrictamente prohibido utilizar bases de datos vectoriales en la nube u otros servicios externos SaaS de memoria que vulneren la confidencialidad del host.
   - Toda información se almacena localmente en montajes Docker relativos a `./data`.

2. **La Triple Capa de Memoria y OS Cognitivo:**
   - **Capa Documental y Contratos (OpenSpec):** Almacenada en `.agent/` (`AGENT.md`, `CONTEXT.md`, `MAP.md`, `STATE.md`, `SPECS.md` y `SYSTEM_PROMPT.md`). El archivo `SPECS.md` rige como el contrato obligatorio antes de codificar.
   - **Capa Operativa (Valkey):** Memoria de corto plazo y caché semántica de consultas.
   - **Capa Semántica (PostgreSQL + pgvector):** Memoria de largo plazo de lecciones aprendidas y embeddings.

3. **Ejecución de Ritos Obligatorios y Heurísticas (ECC + Superpowers):**
   - **Regla obligatoria de PKs:** Todas las llaves primarias (`id`) de tablas nuevas deben ser de tipo `uuid`. No se permite `integer`, `bigint`, `serial` ni `GENERATED BY DEFAULT AS IDENTITY` como PK. Las llaves foráneas que referencien un `id uuid` también deben ser de tipo `uuid`. Usar `DEFAULT uuidv7()` (PG 18+, preferido) o `DEFAULT gen_random_uuid()` (PG ≤ 17).
   - **Rito de Inicio:** Cada vez que el agente sea convocado o se inicie el sistema, debe verificar el estado actual en `.agent/STATE.md`, cargar los contratos activos en `.agent/SPECS.md`, consultar las directrices heurísticas en `.agent/HEURISTICS.md` y verificar la disponibilidad de los servicios locales.
   - **Ciclo de Ejecución Disciplinado (Superpowers):** Todo desarrollo se divide en pasos atómicos, implementados preferentemente con enfoque TDD y validados antes de proceder.
   - **Rito de Cierre:** Al finalizar la interacción, el agente debe actualizar `.agent/STATE.md`, reportar cambios a `.agent/SPECS.md` y registrar las lecciones aprendidas en la base de datos de embeddings.
